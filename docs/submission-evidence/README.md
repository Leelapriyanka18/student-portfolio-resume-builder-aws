# Final Submission Evidence Package

Project: Cloud-Based Student Portfolio and Resume Builder System Using AWS

Purpose: This folder organizes the screenshots, testing proof, AWS proof, GitHub proof, architecture explanation, viva demo flow, and final submission checklist required for B.Tech major project submission.

Scope: Documentation evidence only. Application source code, backend code, frontend code, database schema, AWS resources, and GitHub history are not changed by this package.

## Package Contents

| File | Purpose |
| --- | --- |
| `screenshot-checklist.md` | Complete list of screenshots to capture before final submission. |
| `aws-screenshots.md` | AWS-specific evidence checklist with capture purpose and expected status. |
| `application-screenshots.md` | Application module evidence checklist for UI and workflow screenshots. |
| `github-screenshots.md` | GitHub repository evidence checklist. |
| `testing-evidence.md` | Functional, API, database, AWS, security, and browser testing checklist. |
| `architecture-diagram.md` | Architecture flow and component explanation. |
| `viva-demo-checklist.md` | Five-minute demo, ten-minute demo, and faculty presentation order. |
| `final-submission-checklist.md` | Final readiness checklist for repository, AWS, documentation, report, demo, and viva. |
| `screenshot-index.md` | Numbered screenshot index with names, purpose, and remarks. |

## Recommended Evidence Folder Layout

Store actual screenshot image files in:

```text
docs/screenshots/
```

Use the screenshot names listed in `screenshot-index.md` so that the project report, presentation, and repository evidence remain consistent.

## Submission Use

1. Capture all screenshots listed in `screenshot-checklist.md`.
2. Save screenshots using the exact names from `screenshot-index.md`.
3. Use `aws-screenshots.md` while capturing AWS console evidence.
4. Use `application-screenshots.md` while capturing browser evidence.
5. Use `testing-evidence.md` while recording final test proof.
6. Use `viva-demo-checklist.md` during faculty demonstration preparation.
7. Complete `final-submission-checklist.md` before final upload/submission.

## Final Evidence Rule

Do not fabricate screenshots. Every screenshot should be captured from the actual deployed project, AWS console, GitHub repository, browser, database, or terminal output.
