# Viva Demo Checklist

## Five-Minute Demo Flow

1. Open the S3 static website endpoint.
2. Show Home and About pages.
3. Register a new user.
4. Login with the registered user.
5. Open Dashboard.
6. Save or display Profile data.
7. Save or display Projects data.
8. Save or display Certificates data.
9. Save or display Resume Builder data.
10. Submit Contact form.
11. Logout and show protected route behavior.

## Ten-Minute Demo Flow

1. Introduce the project title and objective.
2. Explain the architecture: Browser, S3, EC2, Spring Boot, RDS, SNS, CloudWatch.
3. Open the deployed S3 website.
4. Show responsive frontend pages.
5. Register a new user.
6. Login and explain JWT authentication.
7. Open Dashboard and explain protected routing.
8. Demonstrate Profile module.
9. Demonstrate Projects module.
10. Demonstrate Certificates module.
11. Demonstrate Resume Builder module.
12. Demonstrate Contact module.
13. Show RDS table evidence for saved data.
14. Show EC2 running instance.
15. Show S3 bucket website hosting.
16. Show CloudWatch logs or metrics.
17. Show GitHub repository structure.
18. Close with project outcome and future scope.

## Faculty Presentation Order

1. Problem statement
2. Project objectives
3. Technology stack
4. AWS architecture
5. Repository structure
6. Live frontend demo
7. Authentication demo
8. Core module demo
9. Database evidence
10. AWS evidence
11. Testing evidence
12. Security explanation
13. Limitations and future scope
14. Final conclusion

## Demo Preparation Checklist

- [ ] S3 website endpoint opens
- [ ] Backend API is running
- [ ] EC2 instance is running
- [ ] RDS instance is available
- [ ] Test user credentials are ready
- [ ] Browser cache cleared or hard refresh completed
- [ ] AWS Console tabs are ready
- [ ] GitHub repository tab is ready
- [ ] Screenshots are available as backup
- [ ] Internet connection verified
